## 简介

插件工厂系统是一个基于 TypeScript 的 Vite 插件开发框架，现已完全迁移到 Vite 官方插件开发模式。该系统直接遵循 Vite 官方指南，使用 HtmlTagDescriptor API 进行插件开发，提供了简洁高效的插件开发体验。

**更新** 系统已移除旧的工厂模式，采用更直接的 Vite 插件开发方式，每个插件都是独立的 Vite 插件实现。

系统的核心特性包括：
- **直接遵循 Vite 官方指南**：使用 HtmlTagDescriptor API 进行 HTML 标签注入
- **简洁的插件架构**：每个插件独立实现，无需工厂模式抽象
- **完整的类型安全保证**：基于 TypeScript 的完整类型系统
- **内置的日志记录和配置验证功能**
- **支持多种插件开发场景**：版本号生成、图标注入、文件复制等

### Vite 原生插件开发模式

**更新** 插件开发已完全迁移到 Vite 官方模式，不再使用工厂函数和抽象基类。

每个插件现在都是一个独立的 Vite 插件实现，直接遵循 Vite 官方插件规范：

```mermaid
sequenceDiagram
participant Dev as 开发者
```

**更新** 移除了 createPluginFactory 函数和 BasePlugin 抽象类，插件开发更加直接和简洁。

### generateVersion 插件

generateVersion 插件展示了如何使用 Vite 官方 API 创建复杂的构建时插件：

**核心功能：**
- **多格式版本号生成**：支持 timestamp、date、datetime、semver、hash、custom 等格式
- **灵活的输出方式**：支持文件输出、代码注入、双重输出
- **智能构建时间处理**：自动获取构建时间并格式化
- **自定义模板支持**：支持占位符模板的自定义格式

**使用示例：**
```typescript
// 基本使用 - 时间戳格式
generateVersion()

// 日期格式
generateVersion({
  format: 'date'
})

// 语义化版本格式
generateVersion({
  format: 'semver',
  semverBase: '2.0.0',
  prefix: 'v'
})

// 自定义格式
generateVersion({
  format: 'custom',
  customFormat: '{YYYY}.{MM}.{DD}-{hash}',
  hashLength: 6
})

// 注入到代码中
generateVersion({
  outputType: 'define',
  defineName: '__VERSION__'
})

// 同时输出文件和注入代码
generateVersion({
  outputType: 'both',
  outputFile: 'build-info.json',
  defineName: '__BUILD_VERSION__',
  extra: {
    environment: 'production',
    author: 'MengXi Studio'
  }
})
```

### injectIco 插件

injectIco 插件完全迁移到 Vite 原生 HtmlTagDescriptor API：

**核心特性：**
- **原生 HtmlTagDescriptor 支持**：使用 Vite 官键 API 进行 HTML 标签注入
- **多种配置方式**：支持字符串配置、完整配置对象、自定义 link 标签
- **文件复制功能**：可选的图标文件复制到目标目录
- **智能 HTML 处理**：自动检测和注入到 HTML 文件的 `<head>` 标签

**使用示例：**
```typescript
// 基本使用
injectIco()

// 使用字符串配置 base 路径
injectIco('/assets')

// 使用完整配置
injectIco({
  base: '/assets',
  icons: [
    { rel: 'icon', href: '/favicon.svg', type: 'image/svg+xml' },
    { rel: 'icon', href: '/favicon-32x32.png', sizes: '32x32', type: 'image/png' }
  ],
  copyOptions: {
    sourceDir: 'src/assets/icons',
    targetDir: 'dist/assets/icons'
  }
})
```

**更新** 插件现在使用 Vite 原生 transformIndexHtml 钩子和 HtmlTagDescriptor API，完全移除了对工厂模式的依赖。

## 性能考虑

### 执行效率优化

**更新** 由于移除了工厂模式和抽象层，插件执行效率得到显著提升：

1. **直接插件调用**：插件直接注册到 Vite，无需额外的工厂包装
2. **按需初始化**：插件只在需要时初始化，减少内存占用
3. **原生 API 利用**：直接使用 Vite 原生 HtmlTagDescriptor API，避免中间层开销

### 内存管理

1. **插件状态隔离**：每个插件维护独立的状态，避免相互影响
2. **生命周期优化**：精确控制插件在构建过程中的执行时机
3. **资源清理**：插件销毁时自动清理相关资源和监听器

### 最佳实践

1. **合理使用日志**：在生产环境中关闭详细日志以提升性能
2. **配置验证**：在开发阶段启用严格验证，生产环境可调整验证级别
3. **错误处理**：根据插件用途选择合适的错误处理策略
4. **插件组合**：合理组合多个插件以避免重复操作

## 故障排除指南

### 常见问题及解决方案

#### 插件未生效

**问题症状：**
- 插件配置正确但没有任何效果
- 控制台没有相关日志

**解决方法：**
1. 检查插件的 `enabled` 配置
2. 验证插件钩子是否正确注册
3. 确认插件在 Vite 配置中的正确位置

#### HtmlTagDescriptor 未注入

**问题症状：**
- 图标标签未出现在生成的 HTML 中
- 插件执行但无输出

**解决方法：**
1. 检查 transformIndexHtml 钩子是否正确返回标签描述符
2. 验证 HtmlTagDescriptor 的配置是否正确
3. 确认插件的执行顺序（order: 'pre'）

#### 版本号生成问题

**问题症状：**
- 版本号格式不符合预期
- 输出文件未生成

**解决方法：**
1. 检查版本号格式配置
2. 验证输出类型设置
3. 确认文件写入权限和路径

### 错误处理策略

插件系统提供了三种错误处理策略：

| 策略 | 行为 | 适用场景 |
|------|------|----------|
| `throw` | 记录错误并抛出异常，中断执行 | 生产环境，需要严格错误处理 |
| `log` | 记录错误但继续执行 | 开发环境，需要完整日志 |
| `ignore` | 记录错误并继续执行，忽略错误 | 特殊场景，允许部分失败 |

## 结论

插件工厂系统已成功迁移到 Vite 官方插件开发模式，提供了更简洁、高效和直接的插件开发体验。

**主要优势：**
- **直接遵循官方规范**：使用 Vite 原生 API，无需额外抽象层
- **简化开发流程**：每个插件独立实现，开发更加直观
- **提升执行效率**：移除工厂模式和抽象层，减少性能开销
- **增强类型安全**：基于 TypeScript 的完整类型系统
- **保持功能完整性**：保留原有插件的所有功能特性

**更新** 新增的 generateVersion 插件提供了强大的版本号管理功能，injectIco 插件完全迁移到 Vite 原生 API，两个插件都展示了现代 Vite 插件开发的最佳实践。

**推荐使用场景：**
- 需要创建 Vite 原生插件的项目
- 希望直接使用 Vite 官方 API 的开发者
- 需要版本号管理和 HTML 标签注入功能的项目
