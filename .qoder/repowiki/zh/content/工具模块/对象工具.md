## 简介
本文件为"对象工具"模块的实用参考文档，聚焦于对象操作工具函数的功能与使用方法，包括：
- 对象属性访问与类型检查
- 数据转换与标准化
- 深度合并与浅层合并
- 过滤与选择
- 序列化与反序列化（JSON）
- 在插件开发中的应用与最佳实践

本模块当前提供两个核心能力：
- 深度合并对象（deepMerge）
- 参数验证器（Validator）

这些能力广泛应用于插件工厂与配置对象的处理，确保插件配置在不同环境与版本下保持一致与可预期的行为。

## 详细组件分析

### 组件一：深度合并（deepMerge）
**更新** 增强了 undefined 值验证、isPlainObject 检查和边界情况处理

- 功能概述
  - 接收任意数量的源对象，按顺序合并到结果对象中
  - 对嵌套对象进行递归合并，使用 isPlainObject 确保可靠的普通对象检测
  - undefined 值会被跳过，不会覆盖已有值
  - 数组会直接覆盖，不会合并
  - null 值会覆盖已有值
  - 返回强类型结果，保证与输入泛型一致

- 关键实现要点
  - 使用 isPlainObject 函数精确检测普通对象（排除数组与 null）
  - 遍历每个源对象的所有自有键，跳过非自有属性
  - 检查 undefined 值并跳过覆盖，保护默认值不被覆盖
  - 对于普通对象且目标也为普通对象时进行递归合并
  - 其他类型（包括数组、null、基本类型）直接覆盖
  - 最终返回合并后的对象

- 适用场景
  - 合并用户配置与默认配置，保护默认值不被 undefined 覆盖
  - 合并多份配置片段，后者优先级更高
  - 在插件工厂中将原始配置标准化为最终运行时配置
  - 处理复杂的嵌套对象结构

- 注意事项
  - undefined 值会被跳过，不会覆盖已有默认值
  - null 值会覆盖已有值，包括 undefined
  - 数组不会被递归合并，而是直接覆盖
  - 使用 isPlainObject 确保只有真正的普通对象才能递归合并
  - 合并结果为新对象，不修改原对象

- 复杂度
  - 时间复杂度：O(N)，N 为所有源对象键的数量
  - 空间复杂度：O(N)，用于存储合并结果

- 使用示例
```typescript
// 基本合并
deepMerge({ a: 1 }, { b: 2 }) // { a: 1, b: 2 }

// undefined 不覆盖
deepMerge({ a: 1 }, { a: undefined }) // { a: 1 }

// 嵌套对象合并
deepMerge({ a: { b: 1 } }, { a: { c: 2 } }) // { a: { b: 1, c: 2 } }

// 数组覆盖
deepMerge({ a: [1, 2] }, { a: [3, 4] }) // { a: [3, 4] }

// null 覆盖
deepMerge({ a: 1 }, { a: null }) // { a: null }

// undefined 跳过
deepMerge({ a: { b: 1 } }, { a: { b: undefined } }) // { a: { b: 1 } }
```

### 组件二：参数验证器（Validator）
- 功能概述
  - 提供流畅的链式 API，对插件配置进行验证
  - 支持必填、类型检查（字符串、布尔、数字、数组、对象）、默认值设置、自定义校验
  - 验证失败时收集错误信息并在 validate() 中统一抛出

- 关键实现要点
  - 字段选择：field() 指定当前校验字段
  - 类型校验：string()、boolean()、number()、array()、object()
  - 其他：required()、default()、custom()、validate()
  - 错误收集：validate() 将所有错误拼接为单一异常抛出

- 适用场景
  - 插件配置的输入校验
  - 在插件工厂中对原始配置进行标准化与默认值填充
  - 为插件使用者提供明确的错误提示

- 使用示例
```typescript
const validator = new Validator(options)
validator
  .field('sourceDir').required().string()
  .field('targetDir').required().string()
  .field('overwrite').boolean().default(true)
  .validate()
```

## 性能考虑
- 深度合并
  - 时间复杂度 O(N)，空间复杂度 O(N)，适合大多数配置合并场景
  - 避免在热路径中对超大对象进行频繁深度合并
  - isPlainObject 检查为 O(1) 操作，性能开销极小
  - undefined 值跳过机制避免不必要的赋值操作
  - 如需浅合并，可自行实现浅拷贝策略（例如使用扩展运算符）
- 参数验证器
  - 验证逻辑线性，开销较小
  - 建议在插件初始化阶段一次性完成，避免重复验证

## 故障排查指南
- 配置验证失败
  - 症状：调用 validate() 抛出包含多条错误信息的异常
  - 排查：确认已先调用 field() 指定字段，再依次调用 required()/类型方法/default()/custom()
- 深度合并未按预期工作
  - 症状：undefined 值意外覆盖、数组未被覆盖、嵌套对象未递归合并
  - 排查：确认使用最新版本的 deepMerge；检查 isPlainObject 检测是否正确；注意 undefined 值会被跳过
- 插件配置未生效
  - 症状：插件未执行或行为异常
  - 排查：检查 enabled、verbose、errorStrategy 等基础配置；核对 copyOptions/incremental 等高级选项

## 附录：API 参考

### 深度合并（deepMerge）
**更新** 增强了 undefined 值验证和 isPlainObject 检查

- 函数签名
  - deepMerge(...sources: Partial<T>[]): T
- 参数
  - sources：可变数量的源对象，类型为 Partial<T>，即允许部分键存在
- 返回值
  - 合并后的对象，类型为 T
- 行为
  - 遍历每个源对象的自有键
  - 使用 isPlainObject 检测普通对象（排除数组与 null）
  - undefined 值会被跳过，不会覆盖已有值
  - 若键值为普通对象且目标也为普通对象，则递归合并
  - 否则直接覆盖（包括数组、null、基本类型）
- 复杂度
  - 时间：O(N)
  - 空间：O(N)
- 使用示例
```typescript
// 基本合并
deepMerge({ a: 1 }, { b: 2 }) // { a: 1, b: 2 }

// undefined 不覆盖
deepMerge({ a: 1 }, { a: undefined }) // { a: 1 }

// 嵌套对象合并
deepMerge({ a: { b: 1 } }, { a: { c: 2 } }) // { a: { b: 1, c: 2 } }

// 数组覆盖
deepMerge({ a: [1, 2] }, { a: [3, 4] }) // { a: [3, 4] }

// null 覆盖
deepMerge({ a: 1 }, { a: null }) // { a: null }
```

- 浅拷贝
  - 仅复制对象的第一层属性，嵌套对象仍为引用
  - 适用于只读配置或不需要修改嵌套结构的场景
- 深拷贝
  - 递归复制整个对象树，生成完全独立的新对象
  - 适用于需要安全修改嵌套结构的场景
- 选择建议
  - 若配置层级较浅且不涉及嵌套对象修改，可使用浅拷贝
  - 若配置包含深层嵌套对象且需要隔离修改，使用深拷贝
