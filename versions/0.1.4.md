# @meng-xi/vite-plugin v0.1.4：环境变量的第一道防线

> 版本：0.1.4 | 协议：MIT | 依赖：Vite ^5.0.0 || ^6.0.0 || ^7.0.0

---

## 写在前面

v0.1.4 的主题是：**让环境变量从"隐式依赖"变成"显式契约"**。

环境变量（`VITE_*`）是连接构建配置与运行时行为的桥梁，但这座桥没有护栏——变量缺失、类型错误、值越界，这些问题往往直到生产环境运行时才暴露。v0.1.4 带来的 `envGuard` 插件，加上 `@common/ui` 终端工具模块和
`@common/validation` 的环境变量验证增强，让你在构建阶段就能拦截问题，甚至把防线延伸到浏览器端。

**本版重点**：

| 能力                      | 一句话说明                              | 你需要做什么 |
| ------------------------- | --------------------------------------- | ------------ |
| `envGuard` 插件           | 环境变量校验 + 模板生成 + 运行时守卫    | 新增配置     |
| `@common/ui` 模块         | ANSI 转义码、Spinner 动画帧、字符串清理 | 按需导入     |
| `@common/validation` 增强 | 环境变量类型/范围/长度验证 + 8 种值类型 | 按需导入     |

**升级方式**：修改 `devDependencies` 中版本号为 `^0.1.4`，无 Breaking Changes，完全向后兼容。

## 一、5 分钟快速上手

### 1.1 安装与最小配置

```json
{
	"devDependencies": {
		"@meng-xi/vite-plugin": "^0.1.4"
	}
}
```

```typescript
import { envGuard } from '@meng-xi/vite-plugin'

export default defineConfig({
	plugins: [
		envGuard({
			required: {
				VITE_API_URL: { type: 'url', required: true }
			}
		})
	]
})
```

### 1.2 立刻看到效果

**校验通过**：

```
✅ [@meng-xi/vite-plugin:env-guard] 环境变量校验通过: 1 个变量全部合法
```

**校验失败**（`VITE_API_URL` 缺失或格式错误）：

```
❌ [@meng-xi/vite-plugin:env-guard] 缺少必需的环境变量: VITE_API_URL
```

构建直接中断。你不用等到部署后才发现问题。

### 1.3 它解决了什么

```typescript
// ❌ 没有 envGuard 的日常
const apiUrl = import.meta.env.VITE_API_URL // undefined → 请求发往 undefined/xxx
const port = import.meta.env.VITE_PORT // 'abc' → parseInt('abc') = NaN
const timeout = Number(import.meta.env.VITE_TIMEOUT) // 1 → 1ms 超时？

// ✅ 有 envGuard：构建时就拦截
envGuard({
	required: {
		VITE_API_URL: { type: 'url', required: true }, // 缺失→中断
		VITE_PORT: { type: 'number', minValue: 1024 }, // 非数字/越界→中断
		VITE_TIMEOUT: { type: 'number', minValue: 1000 } // 越界→中断
	}
})
```

## 二、核心能力：envGuard 三道防线

`envGuard` 从三个层面构建环境变量的防护体系：

```
构建时校验 ──→ .env 模板生成 ──→ 运行时守卫
   (拦截)        (协作)          (兜底)
```

### 2.1 第一道防线：构建时校验

#### 八种值类型

| 类型      | 校验规则                          | 示例值                             |
| --------- | --------------------------------- | ---------------------------------- |
| `string`  | 任意字符串（默认类型）            | `hello world`                      |
| `number`  | 必须为合法数字                    | `3000`                             |
| `url`     | 必须能被 `URL` 构造函数解析       | `https://api.example.com`          |
| `boolean` | `true`/`false`/`1`/`0`/`yes`/`no` | `true`                             |
| `enum`    | 必须在 `enumValues` 列表中        | `production`                       |
| `json`    | 必须能被 `JSON.parse` 解析        | `'{"key":"value"}'`                |
| `semver`  | 必须匹配 `x.y.z` 格式             | `1.2.3`、`2.0.0-alpha.1+build.123` |
| `path`    | 必须以 `./`、`/`、`\` 或盘符开头  | `./src/config.json`                |

#### 范围与长度约束

```typescript
envGuard({
	required: {
		// 数值范围（仅 number 类型）
		VITE_PORT: { type: 'number', required: true, minValue: 1024, maxValue: 65535 },
		VITE_TIMEOUT: { type: 'number', minValue: 1000, maxValue: 60000 },

		// 字符串长度（string / url / path / enum / semver）
		VITE_APP_TITLE: { type: 'string', required: true, minLength: 1, maxLength: 50 },
		VITE_API_KEY: { type: 'string', minLength: 32, maxLength: 128 }
	}
})
```

#### 正则与自定义校验

```typescript
envGuard({
	required: {
		// 正则匹配
		VITE_API_URL: {
			type: 'url',
			required: true,
			pattern: /^https:\/\//,
			message: 'API 地址必须使用 HTTPS'
		},

		// 自定义校验函数
		VITE_CORS_ORIGINS: {
			type: 'string',
			required: true,
			validator: value => {
				const origins = value.split(',')
				const allValid = origins.every(o => {
					try {
						new URL(o.trim())
						return true
					} catch {
						return false
					}
				})
				return allValid || 'CORS 源地址格式错误，多个源以逗号分隔'
			}
		}
	}
})
```

`validator` 返回 `true` 表示通过，返回字符串表示失败且字符串为错误消息。`message` 字段会覆盖所有默认错误消息。

#### 失败处理策略

| failAction | 行为           | 适用场景               |
| ---------- | -------------- | ---------------------- |
| `error`    | 抛出错误，中断 | 生产构建，零容忍       |
| `warn`     | 输出警告，继续 | 开发环境，提醒但不阻塞 |
| `ignore`   | 静默忽略       | 可选变量，仅做记录     |

#### 校验摘要

```
ℹ️ [@meng-xi/vite-plugin:env-guard] 环境变量校验结果: 总计 5 | 通过 3 | 缺失 1 | 失败 1
  ⚠️ [缺失] VITE_API_URL: 缺少必需的环境变量: VITE_API_URL
  ⚠️ [失败] VITE_PORT: 数值 0 小于最小值 1024
```

#### 验证链路

`validateValue` 按以下顺序执行验证，任一步失败即短路返回：

```
缺失检查 → 类型验证 → 范围验证 → 长度验证 → 正则验证 → 自定义验证
  missing   type_error   range_error  length_error  custom_error  custom_error
                                                              全部通过 → pass
```

| 状态            | 含义           | 触发条件                        |
| --------------- | -------------- | ------------------------------- |
| `pass`          | 验证通过       | 所有检查通过                    |
| `missing`       | 必需变量缺失   | 值为空且 `required !== false`   |
| `type_error`    | 类型不匹配     | 值不符合指定类型                |
| `enum_mismatch` | 枚举值不匹配   | 值不在 `enumValues` 列表中      |
| `range_error`   | 数值超出范围   | 超出 `minValue`/`maxValue`      |
| `length_error`  | 长度不符合要求 | 超出 `minLength`/`maxLength`    |
| `custom_error`  | 自定义验证失败 | 正则不匹配或 validator 返回错误 |

### 2.2 第二道防线：.env 模板生成

团队协作中，新人入职第一件事就是问"环境变量怎么配"。`envGuard` 自动生成带注释的 `.env.template` 文件：

```typescript
envGuard({
	generateTemplate: true,
	templateOutput: '.env.template',
	required: {
		VITE_API_URL: {
			type: 'url',
			required: true,
			description: '后端 API 基础地址',
			group: 'API 配置'
		},
		VITE_API_TIMEOUT: {
			type: 'number',
			required: true,
			minValue: 1000,
			maxValue: 60000,
			default: '5000',
			description: 'API 请求超时时间（毫秒）',
			group: 'API 配置'
		},
		VITE_APP_TITLE: {
			type: 'string',
			required: true,
			minLength: 1,
			maxLength: 50,
			description: '应用标题'
		},
		VITE_SENTRY_DSN: {
			type: 'url',
			required: false,
			sensitive: true,
			description: 'Sentry 错误监控 DSN',
			group: '监控配置'
		}
	}
})
```

生成的 `.env.template`：

```bash
# 环境变量模板文件
# 生成时间: 2026-06-02T10:30:00.000Z
# 由 @meng-xi/vite-plugin envGuard 自动生成

# ==============================
# 通用配置
# ==============================

# 应用标题
# [类型: string | 必需 | 最小长度: 1 | 最大长度: 50]
VITE_APP_TITLE=

# ==============================
# API 配置
# ==============================

# 后端 API 基础地址
# [类型: url | 必需]
VITE_API_URL=

# API 请求超时时间（毫秒）
# [类型: number | 必需 | 最小值: 1000 | 最大值: 60000]
VITE_API_TIMEOUT=5000

# ==============================
# 监控配置
# ==============================

# Sentry 错误监控 DSN
# [类型: url | 可选 | ⚠️ 敏感信息]
VITE_SENTRY_DSN=********
```

**模板特性**：

| 特性         | 说明                                            |
| ------------ | ----------------------------------------------- |
| 分组展示     | 按 `group` 字段自动分组，无分组的归入"通用配置" |
| 类型标注     | 每个变量注释包含类型、是否必需、枚举值、范围等  |
| 默认值填充   | 有 `default` 的变量自动填入默认值               |
| 敏感信息隐藏 | `sensitive: true` 的变量值显示为 `********`     |
| 描述信息     | `description` 字段作为变量上方的注释行          |

### 2.3 第三道防线：运行时守卫

构建时校验能拦截大部分问题，但有一种场景它管不了：**用户手动修改了 URL 参数、浏览器扩展注入了变量、或者 CDN 缓存了旧的 HTML**。`runtimeGuard` 把防线延伸到浏览器端。

#### 三种守卫模式

```typescript
envGuard({
	runtimeGuard: true,
	runtimeGuardMode: 'console', // | 'throw' | 'overlay'
	required: {
		VITE_API_URL: { type: 'url', required: true }
	}
})
```

| 模式      | 行为                         | 适用场景                  |
| --------- | ---------------------------- | ------------------------- |
| `console` | 浏览器控制台输出警告         | 开发/测试环境，不阻断用户 |
| `throw`   | 抛出运行时错误，阻止应用启动 | 严格模式，变量缺失即崩溃  |
| `overlay` | 页面顶部显示黄色警告横幅     | 预发布环境，显眼但不阻断  |

#### 守卫代码注入原理

```
Vite 构建 (transformIndexHtml, order: 'post')
    ↓
envGuard 读取构建时校验结果
    ↓
生成 ＜script＞ 标签，包含：
  - 必需变量列表
  - 校验失败的变量列表
  - 对应模式的处理逻辑
    ↓
注入到 ＜/head＞ 标签之前
    ↓
浏览器加载时立即执行
```

注入的代码会在 `window` 上挂载全局对象（默认 `__ENV_GUARD__`）：

```typescript
interface EnvGuardGlobal {
	missing: string[] // 缺失的变量名列表
	invalid: string[] // 校验失败的变量名列表
	allPassed: boolean // 是否全部通过
	data: Array<{
		// 变量元数据
		key: string
		type: string
		description: string
	}>
}
```

你可以在应用代码中读取它：

```typescript
const guard = (window as any).__ENV_GUARD__
if (guard && !guard.allPassed) {
	// 自定义降级逻辑
}
```

#### overlay 模式效果

当 `runtimeGuardMode: 'overlay'` 时，浏览器会在页面顶部显示固定横幅：

```
⚠️ EnvGuard 警告
缺少: VITE_API_URL
校验失败: VITE_PORT
```

横幅样式：黄色背景、深色文字、3px 底部边框、固定定位、最高层级（z-index: 99999）。

### 3.1 @common/ui：终端 UI 工具

提取了终端 UI 相关的通用逻辑，为 `buildProgress` 等插件提供统一的终端输出能力。

**ANSI 转义码工具集**：

```typescript
import { ANSI } from '@meng-xi/vite-plugin/common/ui'

console.log(ANSI.green('✓') + ' 构建成功')
console.log(ANSI.red('✗') + ' 构建失败')

// 光标控制

process.stdout.write(ANSI.clearLine + ANSI.reset + ANSI.green('完成'))
```

| 属性/方法    | 值/签名         | 说明           |
| ------------ | --------------- | -------------- |

| `hideCursor` | `'\x1b[?25l'`   | 隐藏光标       |
| `showCursor` | `'\x1b[?25h'`   | 显示光标       |
| `green`      | `(t) => string` | 绿色文本       |
| `cyan`       | `(t) => string` | 青色文本       |
| `gray`       | `(t) => string` | 灰色文本       |
| `bold`       | `(t) => string` | 粗体文本       |
| `red`        | `(t) => string` | 红色文本       |
| `yellow`     | `(t) => string` | 黄色文本       |
| `magenta`    | `(t) => string` | 品红色文本     |

**Spinner 动画帧**：

```typescript
import { SPINNER_FRAMES } from '@meng-xi/vite-plugin/common/ui'

let frameIndex = 0
const interval = setInterval(() => {
	process.stdout.write(`\r${SPINNER_FRAMES[frameIndex]} Loading...`)
	frameIndex = (frameIndex + 1) % SPINNER_FRAMES.length
}, 80)
```

| 平台    | 动画帧                                  | 原因                           |
| ------- | --------------------------------------- | ------------------------------ |

**ANSI 转义码清理**：

```typescript
import { stripAnsi } from '@meng-xi/vite-plugin/common/ui'

const colored = ANSI.green('hello') + ' ' + ANSI.red('world')

// 计算终端显示宽度
const text = ANSI.bold(ANSI.cyan('状态:')) + ' 完成'
```

### 3.2 @common/validation：环境变量验证体系

v0.1.4 对 `@common/validation` 新增了完整的环境变量验证体系，这些函数不仅服务于 `envGuard` 插件，也可以独立使用。

**验证函数一览**：

| 函数                  | 参数                 | 说明                                            |
| --------------------- | -------------------- | ----------------------------------------------- |
| `validateType`        | `(value, rule)`      | 验证环境变量值的类型                            |
| `validateRange`       | `(value, rule)`      | 验证数值范围（仅 `number` 类型）                |
| `validateLength`      | `(value, rule)`      | 验证字符串长度（仅字符串类类型）                |
| `validateValue`       | `(key, value, rule)` | 验证单个变量（缺失→类型→范围→长度→正则→自定义） |
| `validateEnvironment` | `(env, rules)`       | 批量验证所有环境变量                            |

**独立使用示例**：

```typescript
import { validateValue, validateEnvironment, validateType, validateRange } from '@meng-xi/vite-plugin/common/validation'

// 单个变量验证
const result = validateValue('PORT', 'abc', { type: 'number', required: true })
// { key: 'PORT', status: 'type_error', message: '环境变量值 "abc" 不是合法数字', ... }

// 批量验证
const results = validateEnvironment(
	{ API_URL: 'https://api.example.com', PORT: 'abc' },
	{
		API_URL: { type: 'url', required: true },
		PORT: { type: 'number', required: true, minValue: 1024, maxValue: 65535 }
	}
)
// [
//   { key: 'API_URL', status: 'pass', ... },
//   { key: 'PORT', status: 'type_error', ... }
// ]

// 仅验证类型
validateType('3000', { type: 'number' }) // { valid: true, status: 'pass', message: '' }
validateType('abc', { type: 'number' }) // { valid: false, status: 'type_error', message: '...' }

// 仅验证范围
validateRange('5', { type: 'number', minValue: 1, maxValue: 10 }) // { valid: true, ... }
validateRange('0', { type: 'number', minValue: 1 }) // { valid: false, ... }
```

## 四、实战场景

### 4.1 生产环境：严格校验 + 模板生成 + 运行时守卫

```typescript
import { defineConfig } from 'vite'
import { envGuard, compressAssets, bundleAnalyzer, buildProgress } from '@meng-xi/vite-plugin'

export default defineConfig({
	plugins: [
		buildProgress({ format: 'bar' }),

		envGuard({
			required: {
				VITE_API_URL: { type: 'url', required: true, description: 'API 基础地址', group: 'API' },
				VITE_API_TIMEOUT: { type: 'number', required: true, minValue: 1000, maxValue: 60000, default: '5000', description: '请求超时（ms）', group: 'API' },
				VITE_APP_TITLE: { type: 'string', required: true, minLength: 1, maxLength: 50 },
				VITE_SENTRY_DSN: { type: 'url', required: false, sensitive: true, description: 'Sentry DSN', group: '监控' },
				VITE_LOG_LEVEL: { type: 'enum', enumValues: ['debug', 'info', 'warn', 'error'], default: 'info' }
			},
			failAction: 'error',
			generateTemplate: true,
			runtimeGuard: true,
			runtimeGuardMode: 'console',
			reportOutput: 'env-guard-report.json'
		}),

		compressAssets({ algorithm: 'both' }),
		bundleAnalyzer({ outputFormat: 'json', sizeThreshold: 200 })
	]
})
```

### 4.3 CI/CD 环境：严格校验 + JSON 报告

```typescript
import { defineConfig } from 'vite'
import { envGuard, bundleAnalyzer } from '@meng-xi/vite-plugin'

export default defineConfig({
	plugins: [
		envGuard({
			required: {
				VITE_API_URL: { type: 'url', required: true },
				VITE_APP_TITLE: { type: 'string', required: true }
			},
			failAction: 'error',
			reportOutput: 'env-guard-report.json',
			showSummary: true
		}),

		bundleAnalyzer({
			outputFormat: 'json',
			sizeThreshold: 100,
			gzipSize: true,
			compareWith: 'bundle-analysis-prev.json'
		})
	]
})
```

```bash
# CI 脚本
npm run build

# 检查环境变量校验结果
node -e "
  const report = require('./dist/env-guard-report.json');
  if (!report.allPassed) {
    console.error('环境变量校验失败:', report.results.filter(r => r.status !== 'pass').map(r => r.key + ': ' + r.message).join('; '));
    process.exit(1);
  }
"

# 检查体积告警
node -e "
  const report = require('./dist/bundle-analysis.json');
  if (report.warnings.length > 0) {
    console.error('体积告警:', report.warnings.map(w => w.message).join('; '));
    process.exit(1);
  }
"
```

## 五、进阶：复用验证能力构建自定义插件

`@common/validation` 的环境变量验证函数可以独立使用，不依赖 `envGuard` 插件。以下是一个自定义配置校验插件的示例：

```typescript
import { BasePlugin, createPluginFactory } from '@meng-xi/vite-plugin/factory'
import { validateEnvironment } from '@meng-xi/vite-plugin/common/validation'
import type { Plugin } from 'vite'

interface ConfigGuardOptions {
	rules?: Record<string, import('@meng-xi/vite-plugin/common/validation').EnvFieldRule>
}

class ConfigGuardPlugin extends BasePlugin<ConfigGuardOptions> {
	protected getPluginName() {
		return 'config-guard'
	}

	protected getDefaultOptions() {
		return { rules: {} }
	}

	protected addPluginHooks(plugin: Plugin): void {
		plugin.configResolved = {
			order: 'pre',
			handler: config => {
				this.viteConfig = config
				const env = process.env as Record<string, string | undefined>
				const results = validateEnvironment(env, this.options.rules)

				const failed = results.filter(r => r.status !== 'pass')
				if (failed.length > 0) {
					for (const item of failed) {
						this.logger.warn(`${item.key}: ${item.message}`)
					}
				} else {
					this.logger.success('配置校验通过')
				}
			}
		}
	}
}

export const configGuard = createPluginFactory(ConfigGuardPlugin)
```

### 自动加载 .env 文件

Vite 本身会加载 `.env` 文件，但加载时机在插件 `configResolved` 之后。`envGuard` 的 `autoLoadEnv` 选项确保在执行校验前，所有配置的 `.env` 文件已被解析并注入到 `process.env`：

```typescript
envGuard({
	autoLoadEnv: true,
	envFiles: ['.env', '.env.local', '.env.production', '.env.development'],
	required: {
		/* ... */
	}
})
```

**加载规则**：

1. 按配置的 `envFiles` 列表依次查找
2. 仅注入以 `VITE_` 开头的变量（Vite 约定）
3. 已存在的变量不会被覆盖（先加载的优先级更高）
4. 支持引号包裹的值（`KEY="value with spaces"`）

## 写在最后

v0.1.4 的 `envGuard` 解决的是一个被长期忽视的问题：**环境变量是前端项目中最脆弱的隐式依赖**。它们没有类型声明、没有编译时检查、没有运行时保护。`envGuard` 从三个层面补齐了这块短板：

1. **构建时校验** — 在 `configResolved` 阶段拦截问题，不让错误进入产物
2. **模板生成** — 自动输出带注释的 `.env.template`，降低团队协作成本
3. **运行时守卫** — 将防线延伸到浏览器端，覆盖构建时无法检测的场景

配合 `@common/ui` 和 `@common/validation` 的增强，v0.1.4 让环境变量管理从"靠人"变成"靠工具"。
